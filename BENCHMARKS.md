# AMO-Lean Benchmarks (v2.2.0)

## Criteria

Rubric generated by `/benchmark-qa --strict` during planning.
This section is written ONCE and not modified during execution.

### Correctness
- Roundtrip: `convertBack (convertExpandedSigma a) = a` for all well-formed inputs
- Injectivity: `convertExpandedSigma a = convertExpandedSigma b → a = b`
- convertScalarVar totality: returns `some` for {"x","y","t"}, `none` otherwise
- Pipeline equivalence: amo-lean pipeline produces same C as trust-lean for converted inputs
- 0 sorry, 0 admit, 0 new axiom in all new files

### Performance
- Individual proofs complete in <10s
- Total Bridge module compilation <60s
- No regression in existing amo-lean build time (±15%)
- All conversions linear time O(n)

### Quality
- No `simp [*]` or `native_decide` in proofs
- All public definitions have docstrings
- API minimalism: only `convertExpandedSigma` + `convertBack` + `verifiedCodeGen` exposed
- Module isolation: only `AmoLean.Bridge.TrustLean` imports Trust-Lean
